ТЕХНІЧНЕ ЗАВДАННЯ
Розробка веб-додатку для управління проєктами

1. ОПИС ПРОЄКТУ
Необхідно розробити веб-додаток для управління проєктами з канбан-дошками, призначенням задач та відстеженням прогресу.

2. ФУНКЦІОНАЛЬНІ ВИМОГИ

2.1. Аутентифікація та авторизація
- Реєстрація користувачів
- Вхід в систему
- Відновлення пароля
- Ролі: адміністратор, менеджер проєктів, розробник

2.2. Управління проєктами
- Створення проєктів
- Редагування проєктів
- Видалення проєктів
- Призначення учасників до проєктів

2.3. Канбан-дошки
- Створення колонок (To Do, In Progress, Done)
- Переміщення задач між колонками (drag & drop)
- Додавання задач до колонок
- Редагування задач
- Призначення задач користувачам

2.4. Задачі
- Створення задач з назвою, описом, пріоритетом
- Призначення виконавців
- Встановлення дедлайнів
- Додавання коментарів
- Прикріплення файлів

3. ТЕХНІЧНІ ВИМОГИ

3.1. Frontend
- Framework: Vue.js 3 або Nuxt.js 3
- Мова: TypeScript
- Стилі: Tailwind CSS
- State management: Pinia
- Адаптивний дизайн для мобільних пристроїв

3.2. Backend
- Мова: Node.js з Express або Nest.js
- База даних: PostgreSQL
- API: REST або GraphQL
- Аутентифікація: JWT токени

3.3. Інтеграції
- GitHub API для синхронізації з репозиторіями
- Email сервіс для сповіщень
- Файловий сховище для завантаження файлів

4. ДИЗАЙН ТА UX
- Сучасний та інтуїтивний інтерфейс
- Темна та світла тема
- Швидкий пошук та фільтрація
- Drag & drop для канбан-дошок

5. БЕЗПЕКА
- Хешування паролів
- Валідація всіх вхідних даних
- Захист від XSS та CSRF атак
- Обмеження швидкості запитів (rate limiting)

6. ПРОДУКТИВНІСТЬ
- Оптимізація завантаження сторінок
- Кешування даних
- Lazy loading компонентів
- Оптимізація запитів до БД

7. ТЕСТУВАННЯ
- Unit тести для критичних функцій
- Integration тести для API
- E2E тести для основних сценаріїв

8. ДЕПЛОЙ
- CI/CD pipeline
- Docker контейнеризація
- Моніторинг та логування

9. ДЕДЛАЙН
Проєкт має бути завершений через 3 місяці з моменту початку розробки.

10. БЮДЖЕТ
Орієнтовний бюджет: 150,000 - 200,000 грн
